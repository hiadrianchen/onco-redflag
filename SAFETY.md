# SAFETY.md — OncoRedFlag 安全设计

> 本工具是**临床决策支持的高风险品类**（分诊/就医判断），失败代价是把该急诊的人耽误。
> 安全不是免责声明，而是写进代码与数据流的硬约束。

## 1. 红线（不可违反）

- **不诊断、不判断病因、不解读预后。**
- **不输出"可以居家观察 / 无需处理"**等任何让人安心留家的结论（引擎无 GREEN 级）。
- **不调药、不给替代就医的居家处理方案。**
- **不是急救通道**：所有输出附"危急请拨 120"。
- **禁真实患者数据**：示例模式默认虚构病例；真实使用不持久化、不上传可识别信息、日志脱敏。

## 2. 判级语义（engine/triage.py）

| 级别 | 含义 | 用户侧动作 |
|---|---|---|
| **RED** | 触发需立即处理的红旗 | 立即联系治疗团队 / 急诊（必要时 120） |
| **AMBER** | 低危但化疗期不可忽视（如低热） | 规范复测 + 联系治疗团队咨询 |
| **ESCALATE** | 超范围 **或** 关键信息缺失，无法排除风险 | 升级：联系治疗团队 / 急诊 |
| **NO_REDFLAG** | 未匹配到覆盖的红旗 | **明确不安抚**："未触发 ≠ 安全"，不放心就联系治疗团队 |

## 3. 唯一权威评估顺序（安全语义所在）

> 本节、`engine/triage.py` docstring、`rules/redflags.yaml` 头注释**三处必须字字一致**。
> 以 `engine/triage.py` 的 `evaluate()` 实现为准。

1. **规范化输入**：类型强转，无法解析（如 `"七天前"`）→ None（→ 触发升级），绝不崩溃。
2. **通用急症** RED（意识改变 / 呼吸困难 / 胸痛 / 发绀 / 活动性出血 / 呕血黑便）——**不分人群**先拦。
3. **范围守门（人群）**：儿童 / 免疫 / 靶向 / 放疗 / 其他治疗、血液瘤 / 移植 / 孕产 → ESCALATE（绝不把成人化疗阈值套到他们身上）。
4. **专项红旗**（化疗 0–21 天：发热≥38℃ / 寒战 / 严重呕吐腹泻 / 导管感染）→ RED。
   - 明确红旗**优先于**"信息不足 / 超窗"：不因无关字段缺失而把急症降级。
5. **关键信息缺失**（治疗类型 / 末次化疗天数 / 体温 任一缺失）→ ESCALATE。
6. **窗口守门**：化疗天数 <0 或 >21 → ESCALATE（**超窗发热绝不能落 NO_REDFLAG**）。
7. **AMBER**（37.5–37.9℃ 低热）。
8. **NO_REDFLAG**（明确不安抚；仅"范围内 + 信息齐 + 在窗 + 无任何红旗"时可达）。

> 设计原则：宁可**过度转诊**（over-refer），不可漏判。缺信息 / 超窗 / 脏输入一律往就医方向升级。

## 4. 阈值治理（最关键）

- 所有阈值写在 `rules/redflags.yaml`，**当前全部标 `status: draft-pending-guideline-validation`**。
- 阈值**必须**来自白名单指南（见 `rules/sources.md`），用 KnowS 检索原文固化，并做**中文/本地化复核**。
- **绝不允许 LLM 自行设定或推断阈值**；LLM 只做"自由文本 → 结构化字段"与"结果 → 可读文案"。
- 每次改规则表，必须跑 `tests/danger_cases.yaml` 回归全过；新增红旗须配套危险案例。

## 5. 已知局限

- 仅覆盖少数高频急症信号，**不能覆盖全部危险情况**。
- 仅覆盖**化疗后 0–21 天**窗口；超窗一律升级（ESCALATE），不下"安全"结论。
- 体温阈值默认按**口温**；非口温（腋/耳/额）仅在输出加 `temp_route_note` 提示规范复测，**不下调风险**（不可靠的测量不能用来降级）。
- 发热采用**患者安全口径** ≥38.0℃ 即 RED（不依赖"持续 1 小时"——时长未知不下调风险），比 IDSA 临床 FN 定义更保守。
- intake 抽取可能出错（漏抽症状 / 误抽数值）；脏输入一律强转，失败降为 None → 升级，而非猜测。
- 阈值已用公开指南核证（IDSA 2010 / NCI / ACS，见 `rules/sources.md`）：直接来自指南为 `validated`，定性依据+操作化数值为 `validated-operational`（输出加 `source_note` 提示，避免过度声称循证）。
- 不处理多重共病、用药相互作用等复杂情形——这些一律走 ESCALATE。

## 6. 范围默认假设（重要）

本工具面向**成人实体瘤化疗**患者。`age` / `patient_group` 缺失时按此默认处理——
**不**因缺这两项就升级（否则绝大多数用户都会被升级，工具不可用）；但只要 intake 抽到禁忌信号
（儿童 / 血液瘤 / 移植 / 孕产 / 免疫 / 靶向 / 放疗）即排除并 ESCALATE。

> 因此使用前置说明必须显著告知："若你是儿童/血液肿瘤/移植/孕产患者，或正在做免疫治疗/靶向/放疗，请勿使用本工具，直接联系治疗团队。" 这是 Skill / intake 的强制开场，不是可选项。
