# KnowS 覆盖度金标准评测（T0）

> 目的：在投入开发前，验证 KnowS 对"何时就医 / 红旗阈值"类问题的检索质量，决定主推是否成立。
> 拿到团队 API Key 后逐条跑，**不是"看 API 通不通"，而是看能否支撑确定性规则表的固化**。

## 验收维度（每条 query 都打这 4 项）

1. **权威性**：是否返回**指南 / 患教 / 说明书 / 临床试验**等权威来源（而非论坛、自媒体）？
2. **可量化**：是否含**可落规则的阈值/条件**（如 ≥38.3℃、腹泻≥X 次/日）？
3. **可区分治疗方式**：能否区分化疗 / 免疫治疗 / 靶向（避免把阈值张冠李戴）？
4. **中英文检索**：中文 query 质量如何？必要时中英对照。

判定：①②达标率高 → 主推（红旗规则）成立；若仅擅长文献而非"何时就医"类患教 → 阈值改用本地固化的指南规则表、KnowS 仅作旁证；再不行 → 降级备选 B（问诊准备）。
