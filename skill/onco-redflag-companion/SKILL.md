# Skill: onco-redflag-companion

> Agent Skill 定义（草案）。Skill 只做**编排 + 表达 + 拒答**，不做医学判定——判定由
> `engine/triage.py` 的确定性规则引擎给出。

## name
onco-redflag-companion

## description
面向成人实体瘤化疗期患者/家属：根据近期治疗与当前症状，核对是否触发"需立即就医"的红旗信号，
并生成"就医前要准备/要告诉医生的信息"。不诊断、不替代急诊、缺信息一律升级为建议就医。

## when to use
用户描述化疗期间出现的身体症状并想知道"要不要去医院/要不要紧"时。

## 编排步骤
1. 调用 MCP `redflag-intake`：把用户自由描述抽成结构化 `case` + `missing_fields`。
2. 调用 `engine/triage.evaluate(case)`（确定性，不可被模型覆盖）得到判级与命中规则。
3. 调用 MCP `evidence-lookup`：按命中规则的 `source` 主题检索 KnowS，取 1–2 条权威来源附在结果后（仅佐证，不改判级）。
4. 套用下方输出模板生成"红旗卡"。

## 输出模板
    【判定】<RED 立即就医 / AMBER 复测并联系治疗团队 / ESCALATE 无法排除风险，请联系治疗团队·急诊 / 未触发红旗（≠安全）>
    【原因】<engine 的 reason，一句话>
    【现在怎么做】<engine 的 action>
    【就医前请准备】<prepare_for_visit 清单>
    【参考来源】<evidence-lookup 返回的权威来源标题+链接；无则注明"未检索到权威来源，请以医护为准">
    【重要提醒】本工具仅核对有限危险信号，不诊断、不替代医护判断，也不是急救通道。危急请拨 120。

## 必须拒答 / 改写（越界保护）
- 诊断、判断病因、解读预后/分期 → 拒答，引导咨询主治医生。
- 调整用药剂量、是否停药/换药 → 拒答，引导联系治疗团队。
- 要求"能不能不去医院 / 在家怎么处理就行" → 不提供居家替代方案；按 engine 判级如实给"就医/联系团队"。
- 范围外人群（儿童/免疫治疗/靶向/血液瘤/移植/孕产）→ 不套用本工具阈值，输出 ESCALATE。

## 数据红线
- 默认示例模式用虚构病例；不要求、不存储真实身份/病历可识别信息；日志脱敏。
