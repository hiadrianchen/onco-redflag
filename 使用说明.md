# OncoRedFlag 使用说明

> 化疗期"红旗信号核对 + 就医沟通"助手。只回答一件事:**现在要不要立刻联系治疗团队 / 去急诊**,以及去之前要跟医生讲清楚什么。
> ⚠️ 它**不诊断、不替代医护判断,也不是急救通道**。危急请直接拨 **120**。

## 这是什么形态

- **网页(魔搭 Space)**:浏览器打开链接就能用;手机打开即 H5 体验,无需下载。← 患者/家属用这个
- **MCP 工具**:挂到别的 AI agent 里被调用(无界面)。← 开发者用这个
- 同一套**确定性判定引擎**,三种壳。

---

## 用法一:网页版(患者 / 家属,最常用)

1. 打开 Space 链接（手机/平板/电脑均可；页面右上「**大字模式**」可放大字号，适合长辈）。
2. 填四类信息(知道多少填多少,**不确定就留空**):
   - 治疗类型(选"细胞毒化疗")
   - 末次化疗到今天**几天**
   - 当前**体温**(℃,最好口温)
   - 勾选**当前症状**(寒战/气促/出血/导管红肿…),腹泻次数等
3. 看结果卡的**颜色和结论**,按提示行动。

**结果卡四种颜色的意思:**

| 颜色 | 含义 | 你该做什么 |
|---|---|---|
| 🔴 红 | 触发红旗 | **立即**联系治疗团队 / 去急诊(必要时拨 120) |
| 🟡 黄 | 低热等需关注 | 规范复测 + 联系治疗团队咨询 |
| 🟠 橙 | 信息不足 / 超出覆盖范围 | 无法排除风险,**请就医**(别等) |
| ⚪ 灰 | 未触发红旗 | **不代表安全**;不放心就联系治疗团队 |

> 结果卡还会给一份"**就医前请准备**"清单(末次化疗日期、体温曲线、用药、是否有导管…),照着跟医生说,省时间。

**覆盖范围**:成人实体瘤、化疗后 0–21 天。**不覆盖**:儿童、免疫治疗、靶向、放疗、血液肿瘤、移植、孕产——这些它会直接让你去就医。

## 用法二:作为 MCP 工具(开发者 / 接进别的 AI)

启动 stdio MCP server:

    pip install pyyaml
    python3 mcp/server.py

它暴露两个工具:
- `redflag_check`:输入结构化症状字段 → 返回红旗判定 + 就医沟通卡 + 循证来源(端到端)。
- `evidence_lookup`:按主题检索权威循证来源(provider:local 默认 / knows 在线)。

最小调用示例(JSON-RPC over stdin):

    {"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05"}}
    {"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"redflag_check","arguments":{"treatment_type":"cytotoxic_chemo","days_since_last_chemo":7,"temp_c":38.5,"rigors":true}}}

用 KnowS 在线循证:环境设 `KNOWS_API_KEY` 且 `KNOWS_USE=1`(见 `.env.example`)。

## 关于"循证来源"(KnowS 在线 vs 本地)

- **判定**(红/黄/橙/灰)由确定性引擎 + 已用公开指南核证的阈值决定,**固定可靠**。
- **循证来源**:命中红旗时实时去 **KnowS**(医学文献搜索引擎)检索当前权威指南挂在结果后,让回答有据可依。
- KnowS 不可用 / 无 key → **自动回退本地指南库**,判定不受影响。
- 即:**判断不依赖 KnowS,但"给你看依据"用 KnowS 实时出。**

---

## 安全须知与边界(必读)

- 不诊断、不开药、不给"可以居家观察"的安抚结论;**拿不准一律往就医方向升级**。
- **请勿输入任何真实患者身份 / 病历**;演示与体验一律用虚构病例。
- 阈值依据 IDSA 2010 / NCI / ACS 等公开指南(见 `rules/sources.md`);面向真实患者前仍需 CSCO 中文本地化 + 临床复核。
- 它只核对**有限的**危险信号,**不能覆盖所有急症**。任何疑虑,以医护判断为准;危急拨 120。
